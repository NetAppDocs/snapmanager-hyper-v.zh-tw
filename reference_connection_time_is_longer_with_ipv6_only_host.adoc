= 使用純IPv6主機的連線時間較長
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


* *問題*
+
如果您是在混合式的IPv4和IPv6環境中工作、並將純IPv6主機新增SnapManager 至適用於Hyper-V的支援、則連線可能需要比正常時間更長的時間。

* *原因*
+
之所以會發生此延遲、是因為SnapManager 針對Hyper-V的支援功能會先嘗試使用IPv4傳輸協定。

* *修正行動*
+
若要解決此延遲、請在\windows\system32\drivers\etc\hosts檔案中新增主機。
